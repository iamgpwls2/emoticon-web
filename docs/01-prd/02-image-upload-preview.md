# 이미지 업로드 및 미리보기 기능 명세

## 1. 기능 목적

사용자는 이모티콘 생성을 위해 기준이 되는 이미지를 업로드할 수 있다.  
업로드 전에는 브라우저에서 이미지 미리보기를 확인할 수 있어야 하며, 잘못된 파일을 선택한 경우에는 사용자에게 명확한 오류 메시지를 제공한다.

---

## 2. 사용자 흐름

1. 사용자가 이미지 업로드 영역에서 파일을 선택한다.
2. 프론트엔드에서 파일 확장자, MIME 타입, 파일 크기를 1차 검증한다.
3. 검증을 통과하면 선택한 이미지를 화면에 미리보기로 표시한다.
4. 사용자가 업로드 버튼을 클릭한다.
5. 백엔드 API `POST /api/uploads/image`로 이미지 파일을 전송한다.
6. 백엔드는 로그인 여부와 파일 유효성을 다시 검증한다.
7. 검증을 통과한 이미지는 Supabase Storage의 `user-uploads` bucket에 저장된다.
8. 업로드 성공 시 프론트엔드는 성공 메시지와 업로드 결과(`bucket`, `path` 등)를 표시한다.

---

## 3. 업로드 조건

이미지 업로드는 **로그인한 사용자만** 사용할 수 있다.

허용 파일 형식은 다음과 같다.

| 구분 | 허용 값 |
| -------- | --------------------------------------- |
| 확장자 | `.png`, `.jpg`, `.jpeg`, `.webp` |
| MIME 타입 | `image/png`, `image/jpeg`, `image/webp` |
| 최대 파일 크기 | 5MB |

- multipart field name: `image`
- 업로드 트리거: 파일 선택 후 사용자가 **업로드** 버튼 클릭 (자동 업로드 아님)
- 확장자와 MIME **둘 다** 통과해야 선택·업로드가 진행된다.

---

## 4. 파일 검증 기준

### 프론트엔드 (선택 직후)

사용자가 파일을 선택하는 즉시 다음 항목을 검증한다.

1. 파일이 선택되었는지 확인한다.
2. 파일 확장자가 허용 목록에 포함되는지 확인한다.
3. 브라우저에서 전달된 MIME 타입이 허용 목록에 포함되는지 확인한다.
4. 파일 크기가 5MB 이하인지 확인한다.

검증 실패 시 미리보기를 표시하지 않고 input을 초기화한다.

### 백엔드 (업로드 요청 시)

업로드 요청을 받은 뒤 다음 항목을 다시 검증한다.

1. 요청에 인증 토큰이 포함되어 있는지 확인한다.
2. 인증 토큰으로 로그인 사용자를 확인한다.
3. `image` 필드에 파일이 포함되어 있는지 확인한다.
4. 파일 크기가 5MB 이하인지 확인한다.
5. 파일 확장자와 MIME 타입이 허용 목록에 포함되는지 확인한다.
6. 파일 버퍼의 실제 시그니처(`file-type`)를 확인하여 이미지 파일 위장을 방지한다.

추가 규칙:

- 클라이언트 `originalname`은 저장 파일명에 **사용하지 않는다**.
- Storage 경로의 `user_id`는 토큰에서 검증된 **`req.user.id`만** 사용한다.

---

## 5. 미리보기 기준

프론트엔드는 파일 선택 후 `URL.createObjectURL()`을 사용하여 미리보기 URL을 생성한다.

미리보기는 다음 조건을 따른다.

1. 검증을 통과한 이미지에 대해서만 표시한다.
2. 새 파일을 선택하면 기존 미리보기 URL은 `URL.revokeObjectURL()`로 정리한다.
3. 컴포넌트가 사라질 때도 기존 미리보기 URL을 정리한다.
4. 미리보기는 업로드 완료 여부와 관계없이 **선택한 로컬 파일** 기준으로 표시한다.

UI 참고 (Day 4): `max-height: min(60vh, 360px)`, `object-fit: contain`, `alt="선택한 이미지 미리보기"`

---

## 6. 실패 동작

파일 검증 또는 업로드가 실패하면 사용자가 이해할 수 있는 오류 메시지를 표시한다.

| 실패 상황 | 사용자 메시지 예시 |
| --------------- | -------------------------------------- |
| 파일 미선택 | 업로드할 이미지 파일을 선택해 주세요. |
| 허용되지 않은 확장자 | PNG, JPG, JPEG, WEBP 확장자만 업로드할 수 있습니다. |
| 허용되지 않은 MIME 타입 | 이미지 MIME 타입이 올바르지 않습니다. |
| 5MB 초과 | 이미지는 최대 5MB까지 업로드할 수 있습니다. |
| 로그인하지 않음 | 로그인이 필요합니다. |
| 실제 이미지 파일이 아님 | 실제 이미지 형식이 올바르지 않습니다. |
| Storage 저장 실패 | 이미지 저장 중 오류가 발생했습니다. |

> **Day 4 구현 참고:** 프론트는 확장자·MIME 실패 시 통합 메시지(`PNG, JPG, JPEG, WEBP 형식만…`)를 사용할 수 있다. 백엔드·서비스 계층은 영문 메시지를 반환하며, 프론트에서 한글로 매핑하거나 서버 `error.message`를 그대로 표시한다.

오류 발생 후에도 다른 파일을 다시 선택하거나 업로드를 재시도할 수 있다. 업로드 실패 시 기존 로컬 미리보기는 유지된다.

## 7. 버튼 및 상태 처리

이미지 업로드 UI는 다음 상태를 가진다.

| 상태 | 동작 |
| -------- | --------------------- |
| 기본 상태 | 파일 선택 가능 |
| 파일 선택 완료 | 미리보기 표시, 업로드 버튼 활성화 |
| 업로드 중 | 업로드 버튼 비활성화, 로딩 문구 표시 (`업로드 중…`) |
| 업로드 성공 | 성공 메시지 표시 (`이미지 업로드가 완료되었습니다.`) |
| 업로드 실패 | 오류 메시지 표시 |

업로드 중에는 중복 요청을 막기 위해 업로드 버튼과 파일 선택을 비활성화한다.

## 수용 기준 (AC)

- [ ] 로그인 사용자가 png/jpg/jpeg/webp, 5MB 이하 파일을 선택하면 미리보기가 즉시 표시된다
- [ ] 허용되지 않은 형식·크기 선택 시 오류 메시지가 표시되고 미리보기가 나타나지 않는다
- [ ] 업로드 성공 시 `bucket`, `path`가 반환되고 `uploaded` 이벤트가 발생한다
- [ ] 업로드 중 버튼이 disabled 되고 loading 문구가 표시된다
- [ ] 타 사용자 토큰으로는 다른 user_id 경로에 저장되지 않는다 (backend `req.user.id` 기준)
- [ ] service role key가 프론트·응답·Git에 노출되지 않는다

# 01. Auth (PRD)

## 목적
- Supabase Auth 기반으로 회원가입/로그인/로그아웃을 제공하기 위한 요구사항을 정의합니다.
- Frontend는 **publishable(anon) key만** 사용하며, secret key는 backend 전용임을 전제로 합니다.

## 작성 시점
- 2026-06-02

## 작성할 내용(목차)
- 배경/목표
- 사용자 스토리
  - 회원가입
  - 로그인
  - 로그아웃
  - 세션 유지/자동 로그인
- UX 요구사항
  - 로딩/에러/성공 상태
  - 폼 검증(필수/형식/길이)
  - 접근 제어(보호 라우트)
- 보안 요구사항
  - `VITE_*` 노출 주의
  - secret key/service role은 backend-only
  - 사용자별 데이터 분리(RLS)
- 기능 범위
  - 포함/제외(예: 소셜 로그인 여부)
- 오류 케이스
  - 잘못된 자격 증명, 이메일 미인증(선택), rate limit 등
- 로그/모니터링(선택)
- 수용 기준(AC)

작업 체크리스트:

- [x] 모달이 열릴 때 `body` 또는 page wrapper 스크롤을 잠근다.
- [x] 모달이 닫힐 때 기존 스크롤 상태를 정상 복구한다.
- [x] 모달 내부 메시지 영역만 스크롤되도록 분리한다.
- [x] 마우스 휠, 터치 스크롤, 트랙패드 스크롤 모두 확인한다.
- [x] 모달 닫기, ESC, 외부 클릭 시 scroll lock cleanup이 실행되는지 확인한다.

검증 기준 (후속):

## 3-1. 이미지 업로드 이후 2단계 이모티콘 설정 선택 시 렉 이슈

우선순위: P0

문제:

- 이미지 업로드 이후 2단계 이모티콘 설정 선택 시 렉이 발생한다.

점검 체크리스트:

- [ ] 이미지 미리보기 object URL이 불필요하게 반복 생성되는지 확인한다.
- [ ] `URL.revokeObjectURL()` cleanup이 정상 동작하는지 확인한다.
- [ ] 큰 이미지 업로드 시 렌더링 비용이 과도하지 않은지 확인한다.
- [ ] 설정 선택 시 전체 페이지가 다시 렌더링되는 구조인지 확인한다.
- [ ] computed/watch가 과하게 실행되는지 확인한다.
- [ ] 이미지 파일 객체나 base64 문자열을 reactive state에 과하게 저장하고 있는지 확인한다.
- [ ] 불필요한 console log나 deep watch가 있는지 확인한다.
- [ ] 선택 버튼 클릭 시 API 요청이 중복 발생하지 않는지 확인한다.

개선 체크리스트:

- [ ] 이미지 미리보기는 object URL 기반으로 유지한다.
- [ ] base64 변환이 필요 없다면 제거한다.
- [ ] 설정 선택 상태는 최소한의 primitive 값만 저장한다.
- [ ] 무거운 컴포넌트는 필요할 때만 렌더링한다.
- [ ] 이미지 preview 컴포넌트를 분리해 불필요한 리렌더링을 줄인다.
- [ ] 클릭 이벤트에는 debounce/throttle이 필요한지 검토한다.

검증 기준:

- [ ] 이미지 업로드 후 감정/모션/텍스트 선택 시 렉이 줄어든다.

## 3-2. LLM 채팅 첫 질문 비용 발생 이슈

우선순위: P0
대상 화면: LLM 채팅 모달

문제:

- LLM 채팅 첫 질문이 동적으로 생성되어, 모달을 닫고 다시 열 때마다 비용이 발생할 수 있다.

작업 체크리스트:

- [ ] 모달 open만으로 LLM API가 호출되는지 확인한다.
- [ ] 첫 질문 생성 결과를 세션 상태에 캐싱한다.
- [ ] 같은 이미지/입력값 기준으로 이미 생성된 첫 질문이 있으면 재사용한다.
- [ ] 모달 close 시 채팅 상태를 즉시 초기화하지 않도록 한다.
- [ ] 사용자가 새 이미지 업로드 또는 새 생성 흐름을 시작했을 때만 초기화한다.
- [ ] 첫 질문 생성 트리거를 `모달 열기`가 아니라 `사용자 시작 액션`으로 바꿀 수 있는지 검토한다.
- [ ] API 요청 중 모달을 닫아도 중복 요청이 발생하지 않도록 abort/cancel 처리 검토한다.

검증 기준:

- [ ] 모달을 열고 닫는 동작만으로 LLM API가 반복 호출되지 않는다.
- [ ] Network 탭에서 불필요한 LLM 요청이 보이지 않는다.
- [ ] 새 이미지/새 입력값일 때만 첫 질문이 갱신된다.
- [ ] 기존 채팅 내용이 의도치 않게 초기화되지 않는다.

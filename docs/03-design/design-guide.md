# Design Guide

## 목적
- UI/UX 디자인 기준(컴포넌트, 상태, 접근성)을 정의합니다.
- 기능 PRD를 구현할 때 “일관된 화면 품질”을 보장합니다.

## 작성 시점
- 2026-06-02

## 작성할 내용(목차)
- 디자인 원칙
  - 간결함/명확한 피드백
  - 실패 시 복구 가능(재시도)
- 레이아웃
  - App Shell: Header/Nav + Content
  - 모바일 우선/반응형
- 공통 컴포넌트
  - Button, Input, Textarea, Modal, Toast
  - Spinner/Progress
- 상태 패턴(중요)
  - loading/error/success/empty
  - disabled, optimistic update(선택)
- 폼 UX
  - validation, helper text, error text
- 접근성(A11y)
  - 키보드 내비게이션
  - aria 라벨
- 시각 언어(선택)
  - 색상/타이포/간격 스케일
